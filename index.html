    • Abstract

    The World Wide Web relies on the Hypertext Transfer Protocol (HTTP) to move data between clients and servers. Introduced in the early 1990s, HTTP became the backbone of the web, enabling delivery of pages, media, and APIs at global scale. As the web grew, HTTP evolved to improve speed, security, and reliability—culminating in modern protocols like HTTP/2 and HTTP/3 (over QUIC).

    This site summarizes HTTP’s purpose and traces its major milestones, explaining why each version matters for today’s mobile, streaming, and cloud-first experiences.

    Key Terms

    • HTTP — Application-layer protocol for web requests/responses.
    • HTTPS (TLS) — Encrypted HTTP for confidentiality and integrity.
    • Client–Server — Browser/app requests; server responds.
    • Latency — Delay from request to response.
    • HTTP/2 — Multiplexing and header compression over TCP.
    • HTTP/3 / QUIC — UDP-based transport to reduce latency and avoid HoL blocking.
